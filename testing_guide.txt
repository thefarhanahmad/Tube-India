# Bideo App Testing & Distribution Guide

This guide explains how to share your app with others for testing and how to prepare for production.

## Method 1: Quick Testing with Expo Go (Easiest for internal testing)
This method is best for immediate feedback from friends or team members who have the Expo Go app installed.

1. **Start the Development Server:**
   Run `npx expo start` in your terminal.
2. **Ensure Public Accessibility:**
   Press `s` to switch to "Tunnel" mode. This allows people outside your local Wi-Fi to access the app.
3. **Share the QR Code:**
   - Send a screenshot of the QR code from your terminal.
   - OR copy the `exp://` link and send it via message.
4. **Tester Requirements:**
   - The tester must install the **Expo Go** app (Android/iOS).
   - They scan the QR code or click the link to open your app.

---

## Method 2: Internal Distribution (EAS Build)
This method creates an actual APK (Android) or IPA (iOS) file that testers can install directly. It shows your custom App Icon and Splash Screen.

### Step 1: Install EAS CLI
```bash
npm install -g eas-cli
```

### Step 2: Login to Expo
```bash
eas login
```

### Step 3: Configure Project
```bash
eas build:configure
```

### Step 4: Build for Android (APK)
Modify your `eas.json` to include an APK build profile:
```json
"preview": {
  "android": {
    "buildType": "apk"
  }
}
```
Then run:
```bash
eas build -p android --profile preview
```
Once finished, Expo will give you a link to download the `.apk` file. You can send this file to anyone with an Android phone.

---

## Method 3: TestFlight (iOS) and Google Play Console (Android)
This is the professional way to test, using "Internal Testing" tracks.

1. **Android:** Upload your `.aab` file (generated via `eas build -p android`) to the **Google Play Console** under "Internal Testing."
2. **iOS:** Upload your build to **App Store Connect** and use **TestFlight** to invite up to 10,000 testers via email.

---

## Important Pre-Testing Checklist
1. **API URL:** Ensure `EXPO_PUBLIC_API_URL` in your `.env` is pointing to your hosted backend (e.g., Heroku, Render, AWS) and not `localhost`.
2. **Google Login:** If using Google Login, ensure the Redirect URIs in the Google Cloud Console include the production bundle IDs.
3. **Assets:** Verify that `icon.jpg` and `splash-screen.jpg` look correct in the final build.

## Troubleshooting
- **App Won't Load:** Check if the backend server is running and accessible from the internet.
- **Splash Screen Issues:** If the splash screen doesn't show in Expo Go, it will almost certainly show in a "Method 2" build.
